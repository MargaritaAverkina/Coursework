# Розробка веб-системи розпізнавання дорожніх знаків із використанням класичних методів машинного навчання

Курсова робота з Python (ІСТ-3). Система розпізнає дорожні знаки за фотографією, використовує дві класичні моделі машинного навчання та ансамблевий підхід для підвищення точності.

## Що реалізовано

- попередня обробка зображень із виділенням області знака за кольором (червоний, синій, білий, жовтий)
- дві моделі машинного навчання:
  - HOG + LogisticRegression (вага в ансамблі 62%)
  - Pixels + PCA + KNeighborsClassifier (вага в ансамблі 38%)
- ансамблеве об'єднання результатів із урахуванням згоди між моделями
- оцінка впевненості для кожного передбачення
- переклад назв 58 класів знаків українською мовою
- текстові рекомендації водієві залежно від типу знака
- веб-інтерфейс із підтримкою завантаження файлу та drag & drop
- режим автоматичного тестування на зображеннях із датасету

## Датасет

Використано відкритий датасет **Traffic Sign Dataset for Classification** із платформи Kaggle:
https://www.kaggle.com/datasets/ahemateja19bec1025/traffic-sign-dataset-classification

Датасет містить 58 класів дорожніх знаків. Структура папок:

```
dataset/
  Train/
    0/
    1/
    ...
    57/
  Test/
    000_0001.png
    001_0001.png
    ...
```

Файл `labels.csv` повинен знаходитися в корені проєкту.

## Встановлення та запуск

Встановлення залежностей:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Повне навчання моделей (рекомендовано, займає 10–30 хв):

```powershell
.\.venv\Scripts\python.exe train_models.py --dataset dataset
```

Швидке навчання для демонстрації (займає 1–3 хв, точність нижча):

```powershell
.\.venv\Scripts\python.exe train_models.py --dataset dataset --fast
```

Запуск веб-сервера:

```powershell
.\.venv\Scripts\python.exe -m uvicorn app:app --reload
```

Після запуску відкрийте браузер: http://localhost:8000

## Як працює система

1. Користувач завантажує фото знака через веб-інтерфейс або перетягує його мишею.
2. Сервер отримує байти зображення та передає їх у модуль обробки.
3. Виконується пошук області знака за характерними кольорами (червоний, синій, білий, жовтий).
4. Генеруються три варіанти перегляду зображення для підвищення точності.
5. Для кожного варіанта обчислюються два набори ознак — HOG та піксельні.
6. HOG-модель і Pixel-модель повертають ймовірності для кожного з 58 класів.
7. Ймовірності зважено об'єднуються (HOG 62%, Pixel 38%).
8. Визначається клас із найвищою сукупною ймовірністю.
9. Обчислюється впевненість із урахуванням згоди між моделями.
10. Користувач отримує назву знака, рівень впевненості, топ-3 кандидати та рекомендацію.

## Точність моделей

| Модель | Режим fast | Повне навчання |
|---|---|---|
| HOG + LogisticRegression | ~60% | ~85–90% |
| Pixels + PCA + KNN | ~28% | ~60–70% |
| Ансамбль | ~66% | ~88–92% |
